# プロダクト概要 (product.md)

## 1. プロダクト概要

**Blog AutoPost** は、指定されたブログのRSS/Atomフィードを監視し、新しい記事が
投稿された際に複数のSNSへ自動的に告知を投稿するツールです。CLIに加えて
Web UI も備えており、手動投稿と予約投稿を管理できます。

手動での共有作業を自動化し、コンテンツクリエイターが自身のブログ更新を
効率的にフォロワーへ告知することを支援します。

## 2. コア機能

- **RSS/Atomフィード監視**: 複数のフィードを定期的にチェックし、新着記事を
  自動で検出します。投稿済みの記事は記録され、重複投稿を防ぎます。
- **マルチSNS対応**: X (旧Twitter)、Bluesky、Misskey、Mastodon の4つへ
  投稿できます。
- **複数アカウント対応**: 同一SNS(例: Mastodon)に対して、個人用と
  ビジネス用など複数のアカウントを設定し、それぞれへ投稿できます。
- **投稿先の限定**: `--sns` オプションで投稿先を絞り込めます。`-名前` で
  特定のSNSを除外、`all` で全件を対象にできます。
- **手動投稿**: フィードを介さず、任意のテキストと画像を直接投稿できます。
- **予約投稿**: 指定時刻での投稿を予約し、一覧・追加・変更・削除を
  CLI と Web UI の双方から操作できます。デーモンが定期的に実行します。
- **投稿タイミング管理**: 曜日と時刻の許可リストを設定でき、次に投稿可能な
  枠を自動で算出します。タイミングはSNSごとに個別設定できます。
- **Web UI**: ログイン認証付きのダッシュボードから、即時投稿・時刻指定の
  予約・次のタイミングでの投稿を1つのフォームで操作できます。画像の
  アップロードにも対応します。
- **MCPサーバ**: Web サーバは MCP のエンドポイントを備え、予約投稿の
  一覧取得や追加をAIクライアントから操作できます。
- **画像の自動処理**:
  - 記事のOGP情報などからアイキャッチ画像を自動で抽出します。
  - SNSごとのファイルサイズと解像度の上限に合わせて自動でリサイズします。
  - 添付できるのは画像です。動画や音声の投稿には対応していません。
- **リンクカード**: Bluesky への投稿時に、記事のタイトル・説明・サムネイルを
  含むリンクカードを自動生成します。
- **テキスト最適化**: 各SNSの文字数上限に収まるよう投稿文を調整します。
  X と Mastodon はURLを実際の長さに関わらず一律23文字として数えるため、
  その仕様を吸収した上で文字数を計算します。
- **タグ付与**: 記事のタグをハッシュタグとして付与します。タイトルに
  既出のタグは重複を避けて除外します。

## 3. ターゲットユースケース

- **ブロガー・コンテンツクリエイター**: ブログ記事を更新した際、手動で各SNSへ
  共有する手間を省きたい。
- **メディア運営者**: 運営するWebサイトの新着情報を、複数のSNSチャネルへ
  一括で効率的に配信したい。
- **開発者・技術ブロガー**: 技術ブログの更新を、XやMastodon、Blueskyなどの
  技術者コミュニティへ自動で通知したい。
- **告知の時間帯を選びたい運用者**: 深夜の自動投稿を避けたい、決まった時間に
  まとめて告知したいなど、投稿タイミングを制御したい。

## 4. 主要な価値提案

- **完全自動化**: 一度設定すれば、ブログの更新からSNS投稿までを自動化し、
  運用負荷を大幅に削減します。
- **柔軟な投稿管理**: 自動投稿に加えて、手動投稿・予約投稿・投稿先の限定に
  対応し、CLI と Web UI のどちらからでも操作できます。
- **高い拡張性**: SNSへの投稿は `SnsClient` トレイトで抽象化されています。
  新しいSNSに対応する場合は `src/sns/` にモジュールを追加してこのトレイトを
  実装し、`src/sns/mod.rs` に登録します。コア側のロジックは変更不要です。
- **インテリジェントな投稿**: 文字数調整、画像の自動抽出とリサイズ、
  リンクカード生成など、各SNSの特性を活かした処理を行います。
- **堅牢性**: 各SNSへの投稿失敗は個別に扱われ、他のSNSへの投稿は継続します。
  投稿記録はファイルロックと原子的な書き込みで保護されます。
