# Requirements Document

## Project Description (Input)
レスポンシブデザイン対応をしてください。具体的にはある程度、幅が取れるときには右側に出して確認しやすくしてほしい

## Requirements

### 機能要件

#### 1. レスポンシブレイアウト対応
- **モバイル表示** (幅 < 768px)
  - シングルカラムレイアウト
  - スタックされた要素表示
  - タッチフレンドリーなUI

- **タブレット表示** (768px ≤ 幅 < 1024px)
  - 2カラムレイアウト（必要に応じて）
  - 適応的な余白調整

- **デスクトップ表示** (幅 ≥ 1024px)
  - 右側にサイドバー/詳細パネルを配置
  - 2〜3カラムレイアウト
  - 確認しやすい操作性

#### 2. 右側レイアウト（幅が十分な場合）
- **左側**: 新規投稿フォーム（投稿文、URL、SNS選択、画像アップロード、予約日時）
- **右側**: 予約投稿一覧テーブル（固定配置）
  - テーブルは右側に固定
  - スクロール時も右側パネルが表示状態を保持
  - ソート、フィルタ、一括削除機能も右側に統合
- レイアウト切り替えは自動（メディアクエリ）
- 将来的に設定で左右を入れ替え可能な構造にする（今後の拡張）

#### 3. テーブル対応
- **モバイル**: 横スクロール可能なテーブル
- **タブレット以上**: 通常のテーブル表示
- 行の高さと余白を画面サイズに合わせて調整

#### 4. 入力フォーム対応
- **モバイル**: 全幅入力フィールド
- **デスクトップ**: 複数フィールドを横並び配置（必要に応じて）

#### 5. ナビゲーション対応
- **モバイル**: ハンバーガーメニュー/折りたたみメニュー検討
- **デスクトップ**: 通常のナビゲーション表示

### 非機能要件

#### 1. パフォーマンス
- メディアクエリによる軽量な実装
- レイアウト変更時のリフロー最小化
- CSSのみでの対応（JavaScriptでのDOM操作は最小限）

#### 2. 互換性
- 主要ブラウザ（Chrome, Firefox, Safari, Edge）対応
- iOS/Androidブラウザ対応

#### 3. アクセシビリティ
- レスポンシブ対応後もアクセシビリティを維持
- キーボード操作対応継続
- スクリーンリーダー対応継続

### スコープ外

- 既存の機能追加/削除
- 色やテーマの変更
- 新しいコンポーネントの追加

### 実装対象ファイル

- `src/web/templates/index.html` - CSSメディアクエリ追加

### 成功基準

1. ✅ モバイル（320px）で正常に表示される
2. ✅ タブレット（768px）で読みやすく表示される
3. ✅ デスクトップ（1024px以上）で右側レイアウトが適用される
4. ✅ すべてのブレークポイント間でUI要素が正常に動作する
5. ✅ 既存機能（投稿、編集、削除）がすべてのサイズで動作する
6. ✅ パフォーマンス低下がない

## 状態の訂正 (2026-07-22)

### 何が起きたか

Issue棚卸し(PR #60)にて本specを「完了」と判定したが、**これは誤りだった**。
再調査の結果、2カラムレイアウトは実装されていない。

### 誤判定の原因

判定の根拠としたのが `src/web/templates/index.html` だった。しかしこのファイルは
**配信されていない死にコード**であり、PR #74 で削除されている。

Web UIのルータは `ServeDir::new("static")` を使うため、実際にブラウザへ届くのは
`static/index.html` の方である。削除時点で両者を比較すると、内容が大きく乖離していた。

| 要素 | `src/web/templates/index.html`(死にコード) | `static/index.html`(配信) |
|---|---|---|
| `@media (min-width: 1024px)` | 5箇所 | 0 |
| `@media (min-width: 768px)` | 2箇所 | 0 |
| `@media (max-width: 767px)` | 1箇所 | 0 |

同名のファイルが2箇所に存在するにもかかわらず、どちらが配信されるのかを
`build_router` まで遡って確認しなかったことが原因である。

### 現在の実装状況

`static/index.html` は Tailwind CDN を使用しており、レスポンシブ関連のユーティリティは
以下の4つのみ。いずれも本specが求める「広い画面での2カラム表示」には該当しない。

| クラス | 用途 |
|---|---|
| `md:flex-row` | 投稿方式のラジオボタンを横並びにする |
| `md:p-8` | 余白の調整 |
| `md:text-5xl` | 見出しの文字サイズ |
| `md:w-auto` | ボタン幅の調整 |

`static/index.html` の全履歴を確認したが、`@media` が含まれたことは一度もない。
つまり2カラムレイアウトはRust版のUIへ一度も反映されていない。

### 参考: 実装が残っている場所

死にコード側の実装はコミット `f44f2c7~1` の `src/web/templates/index.html` に残っている。
再実装する場合の参考にできるが、その後 PR #70(テーマ手動切替)や PR #76(テーマ資産の共通化)
などの変更が入っているため、そのまま移植はできない。

### 関連する注意

`docs/responsive_layout_checklist.md` も全項目が完了になっているが、これも死にコード側を
対象とした作業の記録である。実際に配信されるUIの状態は反映していない。
