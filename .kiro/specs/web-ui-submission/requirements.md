# Requirements Document

## Project Description (Input)
blog-autopostに対し、WebアプリケーションとしてのUIと送信機能を追加したいです。
- 認証機能を持ち、実質自身しか使えないようにする
- 以下の内容をSNSに送信できる
	- 文章
		- 送信前に指定SNSの中で送信不可能なものがないか(長すぎなど)をチェックする機構も必要
	- URL(任意)
	- 画像アップロード(任意)
		- 複数ファイル
		- ドラッグ&ドロップ対応
		- 自動リサイズは現ツール内で利用可能だったはず
- 送信時間の設定
	- 即時
	- 指定の日時(分単位、若干のズレは許容)
- ツール自体Pythonベース(uv)を使っているので、これ自体もPythonベースで

## Requirements

### Requirement 1: ユーザー認証
**Objective:** As a ユーザー, I want 自身専用の認証機能を使ってWebアプリケーションにログインしたい, so that 不正なアクセスを防ぎ、安全に利用できる。

#### Acceptance Criteria
1. WHEN ユーザーがWebアプリケーションのルートURLにアクセスした際, IF ユーザーが未認証であれば, THEN a "Webアプリケーション" SHALL ログインページへリダイレクトする。
2. WHEN ユーザーがログインページで有効な認証情報を入力して送信した際, THEN a "Webアプリケーション" SHALL ユーザーを認証し、メインの投稿作成画面へのアクセスを許可する。
3. WHEN ユーザーがログインページで無効な認証情報を入力して送信した際, THEN a "Webアプリケーション" SHALL 認証エラーメッセージを表示し、アクセスを許可しない。
4. WHILE ユーザーが認証済みである間, THE a "Webアプリケーション" SHALL アプリケーション内のページ遷移後も認証状態を維持する。

### Requirement 2: 投稿作成インターフェース
**Objective:** As a ユーザー, I want 直感的なWebフォームを使って、テキスト、URL、画像を組み合わせたSNS投稿を作成したい, so that 効率的にコンテンツを準備できる。

#### Acceptance Criteria
1. WHERE ユーザーが投稿作成ページにいる際, THE a "Webアプリケーション" SHALL 投稿本文を入力するための複数行テキストエリアを表示する。
2. WHERE ユーザーが投稿作成ページにいる際, THE a "Webアプリケーション" SHALL 投稿に含めるURL（任意）を入力するための単一行テキストフィールドを表示する。
3. WHERE ユーザーが投稿作成ページにいる際, THE a "Webアプリケーション" SHALL 複数の画像ファイルをアップロードするためのファイル選択エリアを表示する。
4. WHERE ユーザーが投稿作成ページにいる際, THE a "Webアプリケーション" SHALL ファイル選択エリアへの画像のドラッグ＆ドロップ操作をサポートする。
5. WHERE ユーザーが投稿作成ページにいる際, THE a "Webアプリケーション" SHALL `config.yml`で設定されたSNSアカウントの一覧を選択可能なチェックボックスとして表示する。

### Requirement 3: 投稿前の事前検証
**Objective:** As a ユーザー, I want 投稿を送信する前に、各SNSのルール（文字数制限など）に違反していないか自動でチェックしてほしい, so that 投稿エラーを未然に防ぎ、手戻りをなくせる。

#### Acceptance Criteria
1. WHEN ユーザーが本文テキストエリアに入力する際, THE a "Webアプリケーション" SHALL 選択された各SNSの文字数制限と現在の文字数をリアルタイムに比較・表示する。
2. IF 入力されたテキストが選択されたいずれかのSNSの文字数制限を超過した場合, THEN a "Webアプリケーション" SHALL どのSNSの制限を超えているかを示す警告を即座に表示する。
3. WHEN ユーザーが画像をアップロードした際, THE a "Webアプリケーション" SHALL 既存のメディア検証ロジックを利用して、選択された各SNSの要件（ファイル形式、サイズ、枚数）との互換性をチェックする。
4. IF アップロードされたメディアファイルが選択されたいずれかのSNSの要件を満たさない場合, THEN a "Webアプリケーション" SHALL 具体的なエラー内容（例: 「Xでは動画と画像の同時投稿はできません」）を明確に表示する。

### Requirement 4: 即時投稿機能
**Objective:** As a ユーザー, I want 作成した投稿を「すぐに投稿」ボタンで、選択したすべてのSNSへ一括で送信したい, so that タイムリーな情報発信ができる。

#### Acceptance Criteria
1. WHEN ユーザーが「すぐに投稿」ボタンをクリックした際, IF 投稿内容がすべての選択済みSNSの検証を通過すれば, THEN a "Webアプリケーション" SHALL 選択されたすべてのSNSへの投稿処理を直ちに開始する。
2. WHEN a "Webアプリケーション"がすべてのSNSへの投稿に成功した際, THEN a "Webアプリケーション" SHALL ユーザーに成功メッセージを表示する。
3. WHEN a "Webアプリケーション"が一部または全部のSNSへの投稿に失敗した際, THEN a "Webアプリケーション" SHALL どのSNSで、どのような理由で失敗したかの詳細なエラーメッセージを表示する。
4. WHEN ユーザーが画像付きの投稿を送信した際, THE a "Webアプリケーション" SHALL 既存の画像リサイズ機能を利用して、各SNSの仕様に合わせて画像を最適化してからアップロードする。

### Requirement 5: 予約投稿機能
**Objective:** As a ユーザー, I want 未来の日時を指定して投稿を予約したい, so that 特定のタイミングを狙った計画的な情報発信ができる。

#### Acceptance Criteria
1. WHERE ユーザーが投稿作成ページにいる際, THE a "Webアプリケーション" SHALL 投稿日時を未来に設定するための日付・時刻選択UIを提供する。
2. WHEN ユーザーが未来の日時を指定して「予約投稿」ボタンをクリックした際, IF 投稿内容が有効であれば, THEN a "Webアプリケーション" SHALL 投稿内容、添付メディア、指定された日時を保存する。
3. WHEN a "Webアプリケーション"が予約投稿を正常に受け付けた際, THEN a "Webアプリケーション" SHALL ユーザーに予約完了メッセージを表示する。
4. WHEN システムの現在時刻が予約された投稿の日時（許容誤差内）に達した際, THEN a "Webアプリケーション" SHALL 対応する予約投稿の自動送信処理を開始する。
