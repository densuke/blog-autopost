# Implementation Plan

- [x] 1. Webアプリケーションの基盤構築
- [x] 1.1 依存関係の追加
  - `pyproject.toml`に`fastapi`, `uvicorn`, `python-multipart`, `jinja2`, `apscheduler`を追加します。
  - _Requirements: すべての要件の基礎_
- [x] 1.2 ディレクトリ構造の作成
  - `src/web/`, `src/web/templates/`のディレクトリを作成します。
  - _Requirements: すべての要件の基礎_
- [x] 1.3 基本的なWebサーバーの実装
  - `src/web/main_web.py`にFastAPIのインスタンスを作成し、Uvicornで起動できることを確認します。
  - _Requirements: すべての要件の基礎_

- [x] 2. 認証機能の実装
- [x] 2.1 認証サービスの作成
  - `config.yml`から認証情報を読み取り、ユーザーの資格情報を検証する`AuthService`を実装します。
  - _Requirements: 1.2_
- [x] 2.2 ログイン画面の実装
  - ユーザー名とパスワードを入力するフォームを持つログインページのHTMLテンプレート（`login.html`）を作成します。
  - ログインページを表示するためのFastAPIエンドポイントを実装します。
  - _Requirements: 1.1_
- [x] 2.3 認証処理とセッション管理
  - ログインフォームからのPOSTリクエストを処理し、`AuthService`を呼び出して認証を行うエンドポイントを実装します。
  - 認証成功後、セッションを確立するための仕組み（例: HTTPOnlyの署名付きCookie）を実装します。
  - ログアウト機能を実装します。
  - _Requirements: 1.2, 1.4_
- [x] 2.4 アクセス制御の実装
  - 認証されていないユーザーからのアクセスをログインページにリダイレクトするFastAPIのミドルウェアまたは共通の依存関係を実装します。
  - _Requirements: 1.1_

- [x] 3. 投稿作成UIの実装
- [x] 3.1 投稿フォームのバックエンド実装
  - 投稿作成ページのメインHTMLテンプレート（`index.html`）を作成します。
  - `config_manager`を利用して設定済みのSNSアカウント一覧を取得し、テンプレートに渡してチェックボックスとして描画するエンドポイントを実装します。
  - _Requirements: 2.5_
- [x] 3.2 投稿フォームのフロントエンド実装
  - `index.html`に、本文用のテキストエリア、URL入力フィールド、ファイルアップロードエリアを配置します。
  - リアルタイムの文字数カウンターや、選択されたSNSに応じた文字数制限の警告を表示する基本的なJavaScriptを実装します。
  - _Requirements: 2.1, 2.2, 3.1, 3.2_
- [x] 3.3 画像アップロード機能の実装
  - ファイル選択とドラッグ＆ドロップによる複数画像のアップロードを処理するJavaScriptを実装します。
  - アップロードされた画像をプレビュー表示する機能を追加します。
  - _Requirements: 2.3, 2.4_

- [ ] 4. 即時投稿機能の実装
- [x] 4.1 投稿処理サービスの作成
  - 既存の`media_validator`, `image_resizer`, `text_optimizer`, `plugin_loader`を呼び出し、一連の投稿処理をまとめる`PostingService`を実装します。
  - _Requirements: 3.3, 3.4, 4.4_
- [x] 4.2 即時投稿APIエンドポイントの実装
  - 投稿フォームから送信されたデータ（テキスト、URL、アップロードされたファイル、対象SNS）を受け取る`/api/post`エンドポイントを実装します。
  - _Requirements: 4.1_
- [ ] 4.3 投稿サービスとAPIの統合
  - `/api/post`エンドポイント内で、受け取ったファイルデータを一時的に保存し、`PostingService`を呼び出して投稿処理を実行します。
  - 処理結果（成功または失敗）をJSON形式でフロントエンドに返します。
  - _Requirements: 4.2, 4.3_
- [ ] 4.4 フロントエンドの非同期送信処理
  - 投稿フォームの送信を`fetch` APIを用いて非同期で行い、サーバーからの応答に応じて成功メッセージやエラーメッセージをユーザーに表示するJavaScriptを実装します。
  - _Requirements: 4.2, 4.3_

- [ ] 5. 予約投稿機能の実装
- [ ] 5.1 スケジューリングサービスの初期設定
  - `APScheduler`をFastAPIアプリケーションのライフサイクル（起動・終了）と統合します。
  - 予約ジョブを永続化するため、SQLiteなどファイルベースのジョブストアを設定します。
  - _Requirements: 5.4_
- [ ] 5.2 予約投稿APIエンドポイントの実装
  - 投稿データと予約日時を受け取る`/api/schedule`エンドポイントを実装します。
  - _Requirements: 5.1, 5.2_
- [ ] 5.3 予約ジョブの登録処理
  - `/api/schedule`エンドポイント内で、`APScheduler`に`PostingService`の投稿実行メソッドを指定した日時に呼び出すようジョブを登録します。
  - 予約が成功したことを示すメッセージをフロントエンドに返します。
  - _Requirements: 5.2, 5.3_

- [ ] 6. テストの実装
- [ ] 6.1 ユニットテストの作成
  - `AuthService`の認証ロジック、`PostingService`の各モジュール呼び出し、`SchedulingService`のジョブ登録ロジックに対するユニットテストを作成します。
- [ ] 6.2 統合テストの作成
  - FastAPIのTestClientを使用して、各APIエンドポイント（`/login`, `/api/post`, `/api/schedule`）が正しくリクエストを処理し、適切なレスポンスを返すかの統合テストを作成します。
